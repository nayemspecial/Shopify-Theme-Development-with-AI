### 📌 ১. ভূমিকা: শপিফাই OS 2.0 থিম ডিরেক্টরি স্ট্রাকচার

একটি শপিফাই থিম মূলত কিছু নির্দিষ্ট ফোল্ডার এবং ফাইলের সমষ্টি। শপিফাইয়ের নিজস্ব ইঞ্জিন এই ফোল্ডার স্ট্রাকচার নির্দিষ্ট নিয়মে রিড করে। আমরা যদি কোনো ফোল্ডারের নাম ভুল করি বা ফাইল ভুল জায়গায় রাখি, তাহলে থিম এরর দেখাবে।

চলুন উইন্ডসার্ফ এডিটরে আমাদের প্রজেক্ট রুট ডিরেক্টরিতে শপিফাই OS 2.0-এর স্ট্যান্ডার্ড ফোল্ডারগুলো দেখে নিই এবং এআই-এর সাহায্যে ব্ল্যাংক স্ট্রাকচার তৈরি করি।

### 📌 ২. এআই টুলের সাহায্যে থিম ডিরেক্টরি জেনারেশন

ভিডিওতে স্ক্রিন শেয়ার করে এডিটর ওপেন করুন এবং এআই এসিস্ট্যান্ট (Cascade)-কে এই কমান্ডটি দিন:

> **AI Prompt:** "Create a clean, empty Shopify Online Store 2.0 theme directory structure with all standard folders and a basic config file."

এআই স্বয়ংক্রিয়ভাবে আপনার জন্য নিচের ফোল্ডারগুলো তৈরি করে দেবে। শিক্ষার্থীদের স্ক্রিনে প্রতিটি ফোল্ডারের কাজ ইন-ডিটেইলস বুঝিয়ে বলুন:

* **`assets/`** : এই ফোল্ডারে থিমের সমস্ত সিএসএস (`.css`), জাভাস্ক্রিপ্ট (`.js`), ইমেজ এবং ফন্ট ফাইল থাকবে।
* **`config/`** : এখানে দুটি অত্যন্ত গুরুত্বপূর্ণ ফাইল থাকে:
* `settings_schema.json`: এখানে মার্চেন্টের জন্য গ্লোবাল সেটিংস (যেমন: কালার প্যালেট, ফন্ট, সোশ্যাল মিডিয়া লিংক) এর ব্যাকএন্ড ডিজাইন করা হয়।
* `settings_data.json`: মার্চেন্ট কাস্টমাইজার থেকে যে ভ্যালুগুলো সেভ করবেন, তা এখানে জমা হয়।


* **`layout/`** : থিমের মূল কঙ্কাল বা বেস ফাইল। এর মধ্যে প্রধান ফাইল হলো `theme.liquid`। স্টোরের প্রতিটি পেজ এই লেআউটের ভেতর রেন্ডার হয়।
* **`locales/`** : থিমের বহুভাষিক (Translation) সাপোর্ট ফাইলগুলো (যেমন: `en.default.json`) এখানে থাকে।
* **`sections/`** : থিমের সবচেয়ে পাওয়ারফুল পার্ট। হোমপেজ বা অন্যান্য পেজের রি-ইউজেবল ব্লক বা সেকশনগুলো (যেমন: Header, Footer, Hero Banner) এখানে তৈরি করা হয়।
* **`snippets/`** : ছোট ছোট কোডের টুকরো (যেমন: Product Card, Icon Set) যা বিভিন্ন সেকশনে বারবার ব্যবহার করা যায়। এদের এক্সটেনশনও `.liquid` হয়।
* **`templates/`** : প্রতিটি পেজের (Index, Product, Collection, Cart) মেইন টেমপ্লেট। **জুন ২০২৬ স্ট্যান্ডার্ড অনুযায়ী এই ফোল্ডারের সব ফাইল অবশ্যই `.json` ফরম্যাটে হতে হবে, পুরনো `.liquid` ফরম্যাট ব্যবহার করা যাবে না।**

### 📌 ৩. লিকুইড (Liquid) টেমপ্লেট ইঞ্জিন ফান্ডামেন্টালস

শিক্ষার্থীদের সহজ ভাষায় বোঝান: এইচটিএমএল (HTML) স্ট্যাটিক ডেটা দেখায়, কিন্তু শপিফাই ডেটাবেস থেকে ডাইনামিক ডেটা (যেমন: প্রোডাক্টের নাম, দাম) এনে পেজে দেখানোর জন্য শপিফাইয়ের নিজস্ব ল্যাঙ্গুয়েজ **Liquid** ব্যবহার করা হয়। লিকুইড মূলত ৩টি সিনট্যাক্সের ওপর ভিত্তি করে কাজ করে:

#### **১. আউটপুট সিনট্যাক্স (Output Syntax) — `{{ }}**`

যখন আমরা স্ক্রিনে কোনো ডেটা বা ভ্যারিয়েবল প্রিন্ট করতে চাই, তখন ডাবল কার্লি ব্র্যাকেট ব্যবহার করি।

* **উদাহরণ:**
```liquid
<h1>{{ shop.name }}</h1> <!-- আউটপুট: ProjuktiPlus -->
<p>{{ product.title }}</p> <!-- আউটপুট: টি-শার্টের নাম -->

```



#### **২. লজিক বা ট্যাগ সিনট্যাক্স (Tag Syntax) — `{% %}**`

যখন আমাদের কোনো কন্ডিশন (If/Else), লুপ (For Loop) বা থিম লজিক লিখতে হয়, তখন পার্সেন্টেজসহ কার্লি ব্র্যাকেট ব্যবহার করা হয়। এগুলো স্ক্রিনে সরাসরি কিছু প্রিন্ট করে না।

* **IF Condition উদাহরণ:**
```liquid
{% if product.available %}
  <span class="badge sale">স্টকে আছে</span>
{% else %}
  <span class="badge sold-out">স্টক আউট</span>
{% endif %}

```


* **FOR Loop (প্রোডাক্ট লিস্ট রেন্ডারিং) উদাহরণ:**
```liquid
{% for product in collection.products %}
  <h2>{{ product.title }} - {{ product.price | money }}</h2>
{% endfor %}

```



#### **৩. ফিল্টারস (Filters) — Pipe `|` চিহ্নের ব্যবহার**

আউটপুটের ফরম্যাট বা ডেটা মডিফাই করার জন্য ফিল্টার ব্যবহার করা হয়। ভ্যারিয়েবলের পরে একটি পাইপ (`|`) চিহ্ন দিয়ে ফিল্টারের নাম লিখতে হয়।

* **উদাহরণ:**
```liquid
{{ product.title | upcase }} <!-- নামটিকে বড় হাতের অক্ষরে রূপান্তর করবে -->
{{ product.price | money }} <!-- সংখ্যাকে কারেন্সি বা টাকা/ডলারে কনভার্ট করবে (যেমন: ৳৫০০.০০) -->

```

### 📌 ৪. প্র্যাক্টিকাল ল্যাব: বেসিক `theme.liquid` লেআউট তৈরি

এবার উইন্ডসার্ফে `layout/theme.liquid` ফাইলটি ওপেন করুন এবং এআই-কে বলুন একটি মিনিমাল ভ্যালিড লেআউট জেনারেট করতে। কোডটি স্ক্রিনে লিখে শিক্ষার্থীদের প্রতিটা ট্যাগের গুরুত্ব বুঝিয়ে দিন:

```liquid
<!doctype html>
<html lang="{{ request.locale.iso_code }}">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width,initial-scale=1">
    <title>{{ page_title }} - {{ shop.name }}</title>

    <!-- শপিফাইয়ের প্রয়োজনীয় হেড হুক (বাধ্যতামূলক) -->
    {{ content_for_header }}

    <!-- আমাদের কাস্টম সিএসএস লিংক (assets ফোল্ডার থেকে) -->
    {{ 'theme.css' | asset_url | stylesheet_tag }}
  </head>
  <body>
    
    <header class="site-header">
      <h2>{{ shop.name }}</h2>
    </header>

    <main id="MainContent" role="main">
      <!-- এই হুকের ভেতরেই সমস্ত টেমপ্লেট ও সেকশনের কোড ডাইনামিকালি রেন্ডার হবে -->
      {{ content_for_layout }}
    </main>

    <footer class="site-footer">
      <p>&copy; {{ 'now' | date: '%Y' }} {{ shop.name }}. All Rights Reserved.</p>
    </footer>

  </body>
</html>

```

> **লজিক চেক (Debugging Mindset):** শিক্ষার্থীদের বিশেষভাবে মনে করিয়ে দিন যে, `{{ content_for_header }}` এবং `{{ content_for_layout }}`—এই দুটি হুক ছাড়া শপিফাই থিম কখনোই রান করবে না। শপিফাইয়ের সমস্ত কোর স্ক্রিপ্ট এবং অ্যাপ কোড এই হুকগুলোর মাধ্যমে ইঞ্জেক্ট হয়।
